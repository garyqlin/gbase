# THINKING.md — 先忆后搜，由深到浅

## 思维流程（每条消息都执行）

### 第 0 步：析需求
理解问题是什么、要什么结果、风险多大、值不值得做。
- 别急着动工具、别猜需求、不明确就问。

### 第 1 步：翻笔记 + Knowledge
- **Knowledge 系统已自动加载**（FTS5 匹配你的 user message），先看命中结果。
- **笔记（note_search）要主动翻** — 笔记是深度调研结论（人写的），优先级最高。
- 如果 Knowledge 和笔记都没有 → 进入下一步。

### 第 2 步：读 Session 上下文
当前对话的历史记录。有没有之前的结论、错误或信息？
- 注意：不要被历史错误带偏。如果有矛盾，以第 1 步为准。

### 第 3 步：查 Experience + 进化日志
- **Experience（search_experience）**：已验证的经验模式，告诉你"之前怎么做成的"。
- **进化日志（read_file rules/EVOLUTION.md）**：踩过的坑，告诉你"之前怎么失败过"。
- 先看经验（正向），再看日志（负向），两个一起看不偏颇。

### 第 4 步：查系统确认
- 文件路径、工具可用性、运行状态。
- 用 exec_command / ls / which / lsof。
- 只有前三步找不到确切答案才到这里。

### 第 5 步：上网补充
- search_web，最后一步。
- 防止网络信息带偏判断。

### 第 6 步：闭环写入
重要发现（路径、配置、版本、项目位置）必须写入：
- **Knowledge**：remember_fact
- **Experience**：async_extract_experience（系统自动做）
- **笔记**：note_write
- 不写等于白做。

---

## 核心原则
- **先忆后搜** — 先翻自己知道的，再去系统找，最后上网。
- **由深到浅** — 笔记（深度）→ Knowledge（事实）→ Experience（模式）→ 系统（确认）→ 网络（泛搜）。
- **有结果就停** — 第 1 步找到了就不用走第 4 步，不用每步都跑完。
