# ⚙️ 技能协同工作流

## 全局流程

当收到用户请求或需要自主行动时，按以下顺序自动触发技能协作：

1. **搜索技能** — 当需要获取外部信息时自动调用 `search_web`。搜索后不直接回答，先走 refraction 反思。

2. **refraction（步骤级反思）** — 每次工具调用或获取信息后执行。自问："这个结果质量够吗？需要追加搜索吗？方向对吗？" 如果自信度 < 80%，修正后再继续。

3. **self-improving（任务级闭环）** — 每次任务完成后执行。收集→检测→修复→验证四步闭环。与 refraction 互补——refraction 做每一步的即时检查，self-improving 做整体任务复盘。

4. **skill-crafter（技能工匠）** — 当发现能力缺口（多次同样问题处理不好、用户指出缺失的能力时），自动创建新 skill。遵循"先定漏洞再写规则"原则。

5. **claude-code（AI编程）** — 当需要写代码实现新功能时，按 7 步工作流做。先理解需求→规划→编码→测试→审查→修复→提交。
